About the AI Multi-UE Traffic Classification Project

Project Overview

The project addresses the problem of classifying application traffic in networks with multiple connected User Equipments (UEs). By accurately identifying traffic types, the model enables networks to provide tailored Quality of Service (QoS) for each application.

  • Stacking Ensemble achieves 97.53% accuracy
  • Balanced Accuracy: 97.24%
  • Supports 14 traffic categories (7 standard + 7 VPN variants)
  • Dataset: ~55,000 real network flow samples

Problem Statement

Applications respond differently under varying network conditions:

  • Traffic conditions such as congestion and bandwidth fluctuations
  • Channel states including interference and signal quality
  • Coverage scenarios with mobility, indoor/outdoor environments

Misclassification can reduce QoS and negatively impact user experience. The solution provides real-time, accurate traffic classification for all 14 categories.

Dataset

The dataset contains approximately 55,000 network flow samples with 33 features (13 original + 20 engineered). Traffic categories include browsing, chat, file transfer, mail, P2P, streaming, VoIP, and their VPN counterparts.

Methodology

  • Label encoding and stratified 80-20 train-test split
  • Handling class imbalance with ensemble techniques
  • Feature creation including ratio, log-transformed, and interaction features

Feature Engineering

  • Ratio Features: bytes/packet, packets/duration
  • Log Transformations: log(flowBytesPerSecond, duration, flowPktsPerSecond)
  • Interaction Features: min_fiat * mean_fiat, min_biat * total_biat
  • Total Features: 33 (13 original + 20 engineered)

Model Architecture

The Stacking Ensemble consists of two base learners (XGBoost and LightGBM) and a Logistic Regression meta-learner.

The meta-learner integrates predictions from the base learners to produce the final traffic classification.

Results & Evaluation

  • Overall Accuracy: 97.53%
  • Balanced Accuracy: 97.24%
  • Macro F1 Score: 0.97
  • High precision and recall for all traffic types, near-perfect for P2P and VoIP

Technical Implementation

  • Languages: Python 3.8+
  • ML Libraries: scikit-learn, XGBoost, LightGBM
  • Data Processing: pandas, numpy
  • Visualization: matplotlib, seaborn
  • Deployment: Flask Web Dashboard

Ethical Considerations

  • All traffic data anonymized; no personally identifiable information
  • Focus on flow-level patterns only, not content inspection
  • Class balancing and feature design ensures fairness
